# Tests

These scripts test intent-action alignment using cross-encoder models.
The idea: a cross-encoder compares the user's prompt (`s1`) against a
structured tool call (`s2`) and outputs a score in `[0, 1]`.

- Score **> 0.5** - tool call appears aligned with user intent
- Score **< 0.5** - tool call appears misaligned (potential attack)

Both scripts require Python deps from the project:
```bash
cd agents/simple-langgraph
uv run tests/<script>.py
```

---

## demo_detection.py

Tests 5 simplified scenarios designed for management demos.
Each scenario has one legitimate and one malicious tool call scored
against the same user prompt.

Malicious tools are chosen to be semantically far from user intent
(e.g. user asks to read data, malicious action posts to external webhook)
to maximise detection signal.

```bash
# Public fallback model (no token needed)
uv run tests/demo_detection.py

# With acuvity/intent-action model
HF_TOKEN=<token> uv run tests/demo_detection.py
```

Output: console table + `tests/demo-detection-results.md`

---

## model_comparison.py

Runs a broader comparison across 11 models:
- 5 public cross-encoder models (no auth)
- 6 `acuvity/intent-action` fine-tuned subfolders (requires HF token)

Parses tool calls from `docs/results.md` (generated by `run.sh`) and
full prompts from `prompt-scenarios/advanced-prompts.txt`. Reports
pass rate, average latency, and per-prompt scores for each model.

```bash
# Public models only
uv run tests/model_comparison.py

# All models including acuvity
HF_TOKEN=<token> uv run tests/model_comparison.py
```

Output: `tests/comparison-results.md`

> **Note:** `docs/results.md` must exist before running this script.
> Generate it by running `bash run.sh` with advanced prompts selected.
